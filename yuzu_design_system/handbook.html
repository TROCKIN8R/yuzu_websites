Design System Handbook

Version 1.0 — June 2026. The Yuzu Solutions design language is built on three pillars: Clarity, Zest, and Precision. This handbook is the single source of truth for all brand, product, and marketing surfaces.

🍋

Freshness

Prioritize whitespace and vibrant accents over heavy borders. Let content breathe.

⚡

Agility

Components must feel responsive and provide immediate visual feedback on interaction.

🎯

Precision

Data is our core value. Visualizations must be honest, readable, and accessible.

Color Palette

Electric Yuzu palette with five tones per family. Tone 500 is the canonical brand value. Use lighter tones (100–300) for backgrounds and tints; darker tones (700–900) for text on light surfaces and depth.

Palette Distribution Rule

Maintain "Yuzu Freshness" with a strict 70-15-10-5 hierarchy for accessibility and visual balance.

  • 70% Paper: Foundational whitespace
  • 15% Carbon: Typography and depth
  • 10% Yuzu: Direct focal points
  • 5% Accents: State changes and metrics

Spacing

4px base unit. All margins, padding, and gaps should use tokens from this scale. Never use arbitrary pixel values.

Border Radius

Rounded corners signal approachability. Buttons use --radius-lg (12px). Cards and containers use --radius-2xl (24px).

Layout & Grid

Content max-width is 64rem (1024px). Prose blocks cap at 42rem (672px) for readability. Sidebar documentation layouts use 16rem (256px).

TokenValueUsage

Breakpoints: Mobile < 500px · Tablet < 900px · Desktop ≥ 900px. Grid columns collapse: 5 → 3 → 2 on smaller viewports.

Motion

Animations should feel snappy, not decorative. Default transition duration is 200–300ms with ease curves.

Hover lift: translateY(-2px) + --shadow-lg

Duration: 200ms (micro) · 300ms (standard) · 500ms (emphasis)

Easing: ease for hover · ease-in-out for enter/exit

Accessibility

All color pairings must meet WCAG 2.1 AA contrast ratios (4.5:1 for body text, 3:1 for large text and UI components).
